Smithers 0.25.0 is a big one, around 450 commits since 0.24.2. Here is what
matters most.

## Postgres-backed durability

The durability store now speaks Postgres alongside SQLite and PGlite, with a
schema signature and a DB-backed docs table kept in sync from `.smithers/` by a
file watcher. Backends are resolved explicitly, and a legacy SQLite store can be
migrated with `bunx smithers-orchestrator migrate`. A missing migration fails
loud with `SMITHERS_MIGRATION_REQUIRED` instead of quietly reading stale data.

## A Gateway sync layer

The Gateway gained a batch of read RPCs (`listDocs`, `listAccounts`,
`listPrompts`, `listScores`, `listMemoryFacts`, ticket CRUD, cron listing, and
`getSchemaSignature`) and a TanStack DB sync layer on top: typed collections plus
React hooks like `useGatewayTickets`, `useGatewayScores`, and `useGatewayCrons`,
browser persistence via SQLite-WASM/OPFS, and an optional cloud Electric sync
source.

## Typed workflow outputs

`ctx.output`, `ctx.outputMaybe`, and `ctx.latest` now infer the row type from the
table you pass, so `ctx.outputMaybe(outputs.research, ...).summary` typechecks
instead of handing back an untyped record.

## New agent tools

Workflows get more out-of-the-box capabilities: resilient multi-provider grounded
web search, a generic HTTP tool, Whisper and Deepgram transcription, image
generation, and a document-parsing OCR toolset.

## The real UI is coming

We are gearing up to release the real Smithers UI, so the in-repo
proof-of-concept UI apps (the Cerebras chat PWA, the studio shell, and the demos)
have been removed. Preview the real UI at
[ui-preview.smithers.sh](https://ui-preview.smithers.sh). The custom workflow UI
surface you build with `bunx smithers-orchestrator ui` is unaffected.

## Also in 0.25.0

The `tui` command folded into `up --interactive` and `workflow run --interactive`.
There are roughly 160 fixes across the CLI, gateway, scorers, OpenAPI, and agents,
a green CI, and a hardened release pipeline.

## Upgrade

```sh
bunx smithers-orchestrator@0.25.0
```
