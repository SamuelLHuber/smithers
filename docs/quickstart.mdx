---
title: Quickstart
description: Scaffold and run a Smithers workflow in two commands.
---

```bash
bunx smithers-orchestrator init
bunx smithers-orchestrator init --template idea-to-tickets
bunx smithers-orchestrator workflow run implement --prompt "Add rate limiting"
```

`init` scaffolds `.smithers/` (workflows, prompts, components, agent config). Add `--template <id>` when you want the init result to include the selected starter command and follow-up notes. Run `bunx smithers-orchestrator starters` to browse template IDs.

Inspect the run:

```bash
bunx smithers-orchestrator ps
bunx smithers-orchestrator inspect RUN_ID
bunx smithers-orchestrator logs RUN_ID --tail 20
```

Resume after a crash:

```bash
bunx smithers-orchestrator up workflow.tsx --run-id RUN_ID --resume true
```

Prefer to let your agent drive? [Install the agent skill](/installation#install-the-agent-skill) and just ask for the outcome. Your coding agent runs Smithers for you.

For a full worked example, see the [Tour](/tour). For every CLI command see the [CLI catalog](/cli/overview).
