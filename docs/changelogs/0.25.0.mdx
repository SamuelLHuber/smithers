Smithers 0.25.0 is a large release: roughly 450 commits since 0.24.2. The
headlines are durability on Postgres with a fail-loud migration path, a Gateway
sync layer (TanStack DB collections and React hooks over a batch of new RPCs),
typed workflow output reads, a set of new agent tools, and the removal of the
in-repo proof-of-concept UI apps as we get ready to ship the real UI.

## Breaking changes

- The `tui` command was removed. Run the terminal UI inline with
  `bunx smithers-orchestrator up --interactive` (or `workflow run --interactive`);
  `ps --watch` and `inspect --watch` remain for read-only watching.
- Backends are now resolved explicitly, and a legacy SQLite store can require a
  migration: run `bunx smithers-orchestrator migrate`. A missing migration fails
  loud with `SMITHERS_MIGRATION_REQUIRED` rather than silently reading stale data.
- The proof-of-concept UI packages (`@smithers-orchestrator/smithers`,
  `-studio-2`, `-demo`, `-tui-demo`) were removed (see below). Their previously
  published versions stay on npm.

## Postgres and durable backend

- New Postgres dialect for the durability store alongside SQLite and PGlite, with
  a schema signature and a DB-backed docs table that a file watcher syncs from
  `.smithers/` on disk.
- `bunx smithers-orchestrator migrate` plus backend-aware DB resolution so `ps`,
  `inspect`, and `up` agree on which store they are using.

## Gateway: sync layer and new RPCs

- New RPCs: `getSchemaSignature`, `listDocs`, `listAccounts`, `listPrompts`,
  `listScores`, `listMemoryFacts`, full ticket CRUD
  (`listTickets`/`createTicket`/`updateTicket`/`deleteTicket`), and cron listing.
- TanStack DB collections and matching React hooks (`useGatewayTickets`,
  `useGatewayScores`, `useGatewayMemoryFacts`, `useGatewayCrons`),
  SQLite-WASM/OPFS-persisted collections in the browser, and an optional cloud
  Electric sync source selected with `SMITHERS_SYNC_SOURCE`.

## Typed workflow outputs

- `ctx.output`, `ctx.outputMaybe`, and `ctx.latest` now infer the row type from
  the table argument, so `ctx.outputMaybe(outputs.research, ...).summary`
  typechecks instead of returning an untyped row.

## Agents and tools

- New tool primitives: resilient multi-provider grounded web search, a generic
  HTTP tool, Whisper and Deepgram transcription, image generation, and a
  document-parsing OCR toolset.
- AmpAgent session resume.

## Components and workflows

- New `<Sidecar>` composite component for cheap-model shadow scoring.
- New init-pack workflows: `plan-implement-review-issues`, `bulletproof-audit`
  with `audit-fix-train`, `coverage-codex-swarm`, `verify-push-safety`, and the
  `audit-burndown` validation engine.

## Removed the POC UI apps

We are gearing up to release the real Smithers UI, so the in-repo
proof-of-concept UIs were removed: the Cerebras chat PWA, the studio shell, and
the demos. You can preview the real UI at
[ui-preview.smithers.sh](https://ui-preview.smithers.sh). The custom workflow UI
surface you build with `bunx smithers-orchestrator ui` (via `gateway-react` and
`components`) is unchanged.

## Quality

- Roughly 160 fixes across the CLI, gateway, scorers, OpenAPI, agents, and
  control-plane, a green CI (typecheck and test), and a hardened release pipeline
  whose build drift guard tolerates non-deterministic declaration output.
- The SWE-EVO benchmark gained a native-x86 scorer for emulation-incompatible
  instances, a rate-limit-aware suite supervisor, and an official-leaderboard
  comparison report.

## Upgrade

```sh
bunx smithers-orchestrator@0.25.0
```
