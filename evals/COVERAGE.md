### durability-model

- **Crash-Resume Durability Contract** (`resume: true, --resume flag`) — A completed task is never re-executed. Resume loads persisted state, validates workflow source hash and VCS revision, abandons stale in-progress attempts (>15 min without heartbeat), and continues. · _docs/how-it-works.mdx_
- **Heartbeat Timeout Recovery** (`heartbeatTimedOut(), heartbeatTimeoutMs prop`) — In-progress tasks without a heartbeat for >15 minutes are abandoned on resume as stale and re-executed as a fresh attempt. · _docs/how-it-works.mdx_
- **Hot Reload Mode** (`hot: true, up --hot`) — Edit workflow code during a running execution; changes apply only to newly scheduled tasks while in-flight tasks finish on their original code. · _docs/how-it-works.mdx_
- **Stable Task ID Requirement** (`id prop on Task, Loop, Branch`) — Task IDs must be stable and data-derived across renders. Changing an ID makes it look like a new task; a disappeared ID drops the old task from the plan. · _docs/how-it-works.mdx_
