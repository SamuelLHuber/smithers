## A failing agent is not a failing feature

Some checks need a working coding agent (claude/codex) on PATH and credentials.
If a command fails ONLY because no agent is configured or a provider is out of
credits, mark it `skipped` with that exact reason. Reserve `fail` for an actual
Smithers/CLI regression (a crash, a wrong exit code, a removed-but-documented
command, an unknown-flag rejection, a 404 where routing was promised).
