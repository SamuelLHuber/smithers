# Default onboarding to MDX (incremental adoption); keep TS SDK but not front-and-center

Filed from user feedback on X (onboarding replies), 2026-06-17.

## Problem

*"Default to mdx or something, so that I can adopt incrementally almost as if I'm
using a skill. The ts sdk can ofc remain but should not be front and center."*
MDX lowers the adoption ramp (skill-like, incremental). The TS SDK currently
leads the onboarding.

## Requirement

- Make **MDX the default** authoring format in get-started and examples.
- Keep the TS SDK documented, but move it out of the primary onboarding path
  (a secondary "power user" page).

## Acceptance

- The default get-started path is MDX-first; the TS SDK is reachable but not the
  headline.
